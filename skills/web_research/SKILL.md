# Web Research Skill

## 何时使用

当用户说以下类似的话时激活此 skill：
- "帮我研究…"
- "搜集关于…的信息"
- "调查一下…"
- "找一下…的资料"
- "最新的…是什么"
- "…的现状如何"

## 工作流

1. **提炼关键词**：从用户问题中提取 2-4 个核心搜索关键词，考虑中英文变体
2. **并行搜索**：使用 `web_search` 并行执行多个搜索查询（不同角度/关键词）
3. **筛选结果**：从搜索结果中选取最相关的 3-5 个 URL
4. **抓取内容**：使用 `scrape_webpage` 并行抓取选定页面的详细内容
5. **综合回答**：整合所有来源，给出结构化的综合回答

## 关键原则

- **并行优先**：搜索和抓取尽量并行执行，提高效率
- **多角度验证**：使用不同关键词搜索，交叉验证信息
- **来源标注**：回答中引用具体来源 URL
- **时效性**：优先选择近期发布的内容，注明信息日期

## 输出格式

```
## 研究结论

[2-3句核心结论]

## 详细内容

### [子主题1]
...

### [子主题2]
...

## 来源
- [标题](URL) — 简短说明
- [标题](URL) — 简短说明
```

## 注意事项

- 如果 `scrape_webpage` 返回错误或内容太少，跳过该来源，换下一个
- 搜索结果数量建议设为 8-10（`max_results=8`），然后从中筛选质量最高的
- 对于技术话题，优先抓取官方文档、GitHub、技术博客
- 对于新闻话题，优先抓取权威媒体和官方来源
